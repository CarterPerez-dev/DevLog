function UserStats(): JSX.Element {
  const { user } = useAuthStore();
  const { isSimple } = useTheme();
  const s = isSimple ? simpleStyles : defaultStyles;
  const { data: levelProgress } = useGetLevelProgress();

  if (user === null || user === undefined) {
    return <div className={s.userStats} />;
  }

  const userXP = user.xp ?? 0;
  const userLevel = user.level ?? 1;
  const userCoins = user.coins ?? 0;

  const xpPercentage = levelProgress?.progressPercentage ?? 0;

  return (
    <div className={s.userStats}>
      <div className={s.statItem}>
        <GiDualityMask className={s.icon} />
        <div className={s.content}>
          <span className={s.label}>LVL</span>
          <span className={s.value}>{userLevel}</span>
        </div>
      </div>

      <div className={s.statItem}>
        <GiHawkEmblem className={s.icon} />
        <div className={s.content}>
          <span className={s.label}>XP</span>
          <span className={s.value}>{formatLargeNumber(userXP)}</span>
        </div>
        <div className={s.xpBar}>
          <div
            className={s.xpFill}
            style={{ width: `${xpPercentage.toString()}%` }}
          />
        </div>
      </div>

      <div className={s.statItem}>
        <GiOuroboros className={s.icon} />
        <div className={s.content}>
          <span className={s.label}>COINS</span>
          <span className={s.value}>
            {formatLargeNumber(userCoins)}
          </span>
        </div>
      </div>
    </div>
  );
}

## Security Review

### Security Review for `UserStats` Component

#### Vulnerabilities Found:

1. **Info - Hardcoded Styles:**
   - The styles are hardcoded directly into the component using conditional rendering (`isSimple ? simpleStyles : defaultStyles`). While this is not a direct security issue, it could lead to issues if the stylesheets contain sensitive information.
   - **Fix:** Ensure that no sensitive data is stored in stylesheets. Consider using environment variables or a configuration management system.

2. **Info - Potential XSS:**
   - The `formatLargeNumber` function is used without any sanitization or escaping, which could lead to Cross-Site Scripting (XSS) if the input contains malicious content.
   - **Fix:** Sanitize and escape user inputs before rendering them in the DOM.

3. **Info - Potential TOCTOU Bug:**
   - The `useGetLevelProgress` hook might fetch data asynchronously, but there is no check to ensure that the fetched data is not outdated or stale.
   - **Fix:** Implement a mechanism to verify the freshness of the fetched data before using it in calculations.

4. **Info - Error Handling:**
   - There are no error handling mechanisms for potential failures in fetching user data or level progress.
   - **Fix:** Add try-catch blocks and handle errors gracefully, possibly displaying a fallback message or loading state.

#### Overall Security Posture:

The component is relatively secure but could benefit from additional checks to prevent XSS and ensure data freshness. Hardcoded styles should be reviewed for any sensitive information. Implementing proper error handling will also improve the overall security posture of the application.
